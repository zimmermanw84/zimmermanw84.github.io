NoSQL

Phase 0. Week 8 - Thursday, January, 2015

Welcome back to my blog! During this installment we're going to be discussing NoSQL databases (DBs). Here is a very helpful VIDEO that help me understand what NoSQL is. By looking at the name you would think that NoSQL is as it sounds, but it's actually "Not only SQL". This is another type of database management system.

So why would I not use a relational database?
One instance would be when you have a massive amount of data and want to maintain a level of performance. When using a relational DB it can get messy with many, many to many relationships. That doesn't bode well for performance, joining tables and making complex queries could really bog you down and slow the DB down. Also, what if the data is not structured? (Like this blog) You would probably go for a NoSQL type DB.

Sounds great right? So when wouldn't I want to use one?
Well, whenever you need the DB to preform complex transactions would be one instance. Anytime you would be putting more pressure on the DB to preform tasks rather then just storing is probably when you would think about not using a NoSQL DB.

In conclusion, if you need the DB to provide constraints and/or preform complex transactions you probably would not choose a NoSQL type data structure. However, if your primary focus is horizontal scalability and fast performance, then NoSQL is probably what you're looking for. I highly suggest you watch the video link above. It offers great insight into this subject. Thanks for reading!!
