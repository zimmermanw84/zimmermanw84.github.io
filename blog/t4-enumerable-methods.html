Enumerable#map

Phase 0. Week 4 - Thursday, November 20th, 2014

Welcome back to my blog, you are much appreciated. In this installment we are going to talking about an enumerable method called map. If you don't know what a method is don't worry. A method is the equivalent of a verb in the english language. Methods are your action words, methods do stuff. K, so what is an enumerable? Right, well...enumerable is a type of module. K, whats a module? Sense the scope of this particular blog post is about the map method I am not going to go into great detail about modules, but basically a module is a toolbox that holds methods. So back to business...

Map is a very useful method that I used recently, without knowing I was going to be writing a blog post about it. The map method is known as an iterator, all that means is that it is going to implement the following code block on the object you called the map method on. Here is an example on how I used map:

[1,"cat", 3, "dog"].map! do |num| if num.is_a?(Fixnum) num + 1 else num end =>> [2, "cat", 4, "dog"]

So first thing you probably noticed was that I used map! not map. The difference is pretty common in Ruby. The (!) usually indicates that it is a destructive method, actually changing the original rather than spitting out a modified duplicate or clone. In this specific instance I wanted to modify the original so I used map!. I am using map! to iterate through an array and checking to see if each number is a Fixnum. If it is then the code adds 1 to it and if it's anything else, like a string, it returns it unchanged.

The magic is in the map! method. It's running the code block on each element in the array returning the modified value, if the condition is met to be modified.
