Javascript v. Ruby

Phase 0. Week 7 - Wendnesday, December 10th, 2014

Welcome back once again to my technical blog. In this segment I am going to be discussing the difference between the Javascript and Ruby programing languages. We could go on and on about the differences of the two, but we are going to narrow our scope for this blog. We're going to focus on Ruby classes and Javascript constructor functions. The resemblance between the two are glaring, so thats where we will start. Below, I provided an example of a program that does the same thing so you can draw a comparison visually.

We'll go ahead and look down at the bottom of the examples given, at the driver test code. It looks almost identical. You could have an intimate knowledge of, lets say Ruby, and look at the Javascript test code and surmise that you are creating an instance of an object, dog in this case, and setting it to a variable chewy. Then you are telling it to speak, then changing the name to "Walt", then speaking again. From the same perspective, you could look over the lines 1-11 and get an idea of what it's doing, and you could also surmise that the logic is the same. But that syntax though...

The major difference between the two languages is mainly the syntax. Starting off by learning ruby, the syntax in Javascript can be a little intimidating. All those () and }}. Ruby, in my opinion, is a lot more intuitive as far as the syntax is concerned. But the major difference between the class and constructor function is the names of them. Because Ruby is designed to be OO (Object Oriented), the syntax is designed for class objects and defining them as such. Javascript can be OO but also has a lot of functional design features. So the major difference again is the vernacular. There is no such keyword in Javascript for class but using the constructor function, essentially, you are doing the same thing. But what about the methods?

What about those methods? Again the same thing is happening. In the Javascript example you are using the constructor function to create a prototype of dog. Within every instance of dog you are creating a function that can only be used on instances of dog called speak. Under the constructor function we add to the prototype, the changeName function so you can change your dogs name if you want to. Now if you look over to the Ruby example, there's the same speak method (basically a function). That speak method can be called on all instances of the dog class. But in the Ruby example we use the built in magic of Ruby to create our name change method. The magic of the attribute accessor is built in for ruby, allowing you to change/read/write attributes of your class.

In conclusion, the biggest difference is syntax. Also, Ruby was designed with OO programing in mind so there are more shortcuts like our attribute accessor for class objects. But we're doing the same thing in Javascript, creating a prototype of dog so we can create instances of it. It just doesn't look as good :)
